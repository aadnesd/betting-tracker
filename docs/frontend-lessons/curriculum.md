# Frontend Development Daily Curriculum

This curriculum is tuned for this repository: Next.js App Router, React, TypeScript, Tailwind CSS, shadcn/Radix-style components, Playwright, and matched-betting product workflows.

Each lesson is intentionally small. Read the concept, inspect the referenced files, then complete the exercise in 20-40 minutes.

## 30-Day Path

1. Mental model: HTML, CSS, JavaScript, React, and Next.js responsibilities.
2. Read a component: props, JSX, imports, exports, and render flow.
3. Styling basics: utility classes, spacing, typography, and responsive constraints.
4. Layout: flex, grid, gap, alignment, and stable dimensions.
5. Design systems: reuse `components/ui/*` before inventing new controls.
6. Client vs server components: when `"use client"` is required.
7. State: local state, controlled inputs, and derived values.
8. Forms: labels, validation, errors, pending states, and submission shape.
9. TypeScript for UI: component props, unions, nullable data, and narrowing.
10. Data display: tables, badges, summaries, and scan-friendly formatting.
11. Loading and empty states: skeletons, disabled controls, and progressive rendering.
12. Error states: recoverable failures, user copy, and retry affordances.
13. Accessibility: keyboard flow, labels, focus, contrast, and semantic structure.
14. Responsive UI: mobile-first constraints without viewport-scaled text.
15. App Router: route folders, layouts, pages, route groups, and server actions.
16. Data fetching boundaries: server queries, client interactions, and cache shape.
17. Domain UI: model matched-betting concepts clearly in the interface.
18. Component extraction: when duplication is helpful signal vs abstraction pressure.
19. Charts and visual summaries: make data legible before making it decorative.
20. Dialogs and sheets: focus management, escape routes, and destructive actions.
21. Optimistic UI: latency, rollback, and user trust.
22. Playwright basics: locate by role, test happy path, test failure path.
23. Visual verification: screenshots, mobile viewport checks, and text overflow.
24. Performance: bundle awareness, render cost, and avoiding unnecessary client code.
25. Motion: restrained feedback, transitions, and when animation helps.
26. Design polish: rhythm, hierarchy, alignment, and reducing visual noise.
27. Refactoring UI safely: preserve behavior while improving structure.
28. Production readiness: lint, format, build, and test expectations.
29. Feature slice: plan and implement a small UI improvement end to end.
30. Review: explain the tradeoffs in your own code and identify the next skill gap.
